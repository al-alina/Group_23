GitHub. HW_2
1. На локальном репозитории сделать ветки для:
- Postman
- Jmeter
- CheckLists
- Bag Reports
- SQL
- Charles
- Mobile testing

git branch branch_name

2. Запушить все ветки на внешний репозиторий
git push -u origin branch_name

3. В ветке Bag Reports сделать текстовый документ со структурой баг репорта
git checkout BugReports
touch BugReport.txt
vim BugReport.txt
i

Bug_id:
Summary:

Priority:
Severity:

Environment:

Preconditions:

Steps to reproduce (STR):

Expected Result (ER):
Actual Result (AR):

Attachments:

:wq

4. Запушить структуру багрепорта на внешний репозиторий
git commit -am "adding BugReport"
git push

5. Вмержить ветку Bag Reports в Main
git checkout main
git merge BugReports

6. Запушить main на внешний репозиторий.
git push

7. В ветке CheckLists набросать структуру чек листа.
git checkout CheckLists
touch Checklist.txt
vim Checklist.txt
i

#	Priority		Functionality to be checked	Status 	   	Comment
	(optional)
1.	High/Medium/Low		Check...			Pass/Fail			

:wq

8. Запушить структуру на внешний репозиторий
git commit -am "Adding Checklist"

9. На внешнем репозитории сделать Pull Request ветки CheckLists в main
press "create pull request"
comment: merging CheckLists_branch
press "merge pull request"
press "confirm merge"

10. Синхронизировать Внешнюю и Локальную ветки Main
git checkout main
git pull
